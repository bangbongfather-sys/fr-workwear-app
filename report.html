🔧
진단 — 수정 필요 사항
Critical localStorage 5MB 한도 초과 위험
현재 50개+ localStorage 키에 제품 이미지(base64), 장부, 매입 이력, 견적 이력 등 대용량 데이터를 저장 중입니다. IndexedDB로 이미지는 이미 분리했지만, 나머지 데이터가 누적되면 쓰기 실패(QuotaExceededError) 시 사용자가 모르는 사이에 데이터가 저장 안 될 수 있습니다. 현재 save() 함수는 실패 시 console.warn만 합니다.
✅ 수정안: localStorage 사용량을 실시간 모니터링하는 상태 표시기 추가. 80% 초과 시 경고 배너 표시. Firebase/Cloudflare 클라우드 동기화를 메인 스토리지로 올리고 localStorage는 캐시로 강등.
Critical API 키·Drive 토큰 localStorage 노출
Claude API 키(frw_apikey), Google Drive OAuth 토큰(frw_drive_token), Gmail 토큰(frw_gmail_token)이 localStorage에 평문 저장됩니다. 같은 기기를 다른 사람이 사용하거나 XSS 공격 시 탈취 가능합니다.
✅ 수정안: Claude API 호출은 Cloudflare Worker 프록시로 서버사이드화. OAuth 토큰은 httpOnly 쿠키 또는 sessionStorage + 자동 갱신 구조로 변경.
Warning 17,441 라인 단일 파일 — 성능·유지보수
모든 기능이 index.html 한 파일에 집중되어 있어 초기 로딩 시 전체 파싱이 필요합니다. Vite/React 빌드 시스템으로 전환하면 코드 스플리팅, 트리쉐이킹으로 초기 로드 60%+ 개선 가능합니다. 현재 파일 수정 시 의도치 않은 영역 충돌 위험도 높습니다.
✅ 수정안 (Lovable 적합): Vite + React로 마이그레이션. 탭별 lazy import 적용.
Warning 모바일 UX — 복잡한 테이블/입력폼 미최적화
viewport 메타태그·기본 media query는 설정되어 있지만, 장부·매입현황·견적서 작성 등 복잡한 테이블이 모바일에서 가로 스크롤만으로 처리됩니다. 하단 네비게이션(mobile-bottomnav)이 선언되어 있지만 실제 렌더링 조건이 불명확합니다.
✅ 수정안: 768px 이하에서 카드형 레이아웃 전환. 하단 탭 바 활성화. 입력 폼에 large hit target(최소 44px) 적용.
Warning 자동 저장 커밋 메시지 — 변경 추적 불가
"auto: 2026-06-08 11:09" 형태의 자동저장 커밋이 실제 작업 내용을 숨깁니다. 채팅이 끊기면 어디까지 작업했는지 파악하는 데 시간이 걸립니다. (이번 세션 재개 시 직접 경험한 문제)
✅ 수정안: auto-commit 메시지에 변경된 함수명·섹션 포함. 또는 세션 시작 시 이전 auto 커밋들을 squash하여 의미 있는 커밋으로 정리.
Info 오프라인 지원 없음
현재 네트워크 없이는 Firebase 동기화, AI 견적서 생성, 나라장터 크롤링, 뉴스 피드 모두 불가합니다. Service Worker를 추가하면 기본 입력/조회는 오프라인에서도 가능합니다.

✅ 수정안: 설정 페이지에 "전체 데이터 JSON 내보내기" 버튼 + 백업 타임라인 복원 UI 추가.
